# cooL UI 0.2.0

cooL UI 0.2.0 是仅供本地使用的 Alpha 快照，不发布到公共仓库。本版本把共享契约扩展到 43 个组件，同时如实保留各平台的保守成熟度标记。

## 主要变化

- MonthCalendar 在 SwiftUI、Compose、ArkUI 和微信小程序中均为受控组件；选中日期、展示月份和日期数据由调用方拥有，Catalog 覆盖极端状态。
- 微信 Button 改为渲染原生 `<button>` 并转发受支持的平台能力。这是有意为之的 0.x 破坏性改进，DOM、表单归属和事件行为均与旧版外壳实现不同。
- 微信 TabBar 与 SegmentedControl 使用受控值，通过事件请求父级更新，不再直接修改内部选中状态。
- `contracts/release.json` 是 package 版本、生成源码、Catalog 标题、本地制品和文档的唯一发布版本源。
- 修正 ArkUI ARGB 通道解析和选中日期的对比度。

## 本地交付与成熟度

SwiftPM 使用仓库检出目录，Compose 使用本地 Maven 仓库，微信使用生成的 npm tarball，ArkUI 使用本地组装的 HAR。HarmonyOS 6 HAR 必须在真实 DevEco 环境中完成构建；源码验证不能替代原生构建结果。

Changesets 只记录变更意图。0.2.0 已由唯一发布契约完成版本同步，因此本说明不会添加错误推进到 0.3.0 的待处理版本变更。
