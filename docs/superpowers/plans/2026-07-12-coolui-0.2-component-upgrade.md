# cooL UI 0.2 Component Upgrade Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver cooL UI `0.2.0` as a locally installable, four-platform component library with a controlled `MonthCalendar`, native WeChat button semantics, controlled WeChat tab and segment controls, synchronized bilingual documentation, and reproducible artifacts—without changing the consuming mini-program.

**Architecture:** Keep `contracts/components.json` as the component capability source and introduce `contracts/release.json` as the single release-version source. Generated registries, documentation, and generic components continue to come from contracts, while components with platform-specific behavior use explicit generator templates or focused native source files so regeneration is deterministic. `MonthCalendar` accepts precomputed day cells and emits selection/month-direction events; it never owns calendar, lunar, holiday, or scheduling logic.

**Tech Stack:** Node 22, pnpm 10, Turbo, Node Test Runner, Swift 6.2 / SwiftUI iOS 26, Kotlin 2.3 / Jetpack Compose / Android API 31+, ArkTS / ArkUI HarmonyOS 6, native WeChat Mini Program components for base library 3.14.3, VitePress, Changesets, GitHub Actions.
