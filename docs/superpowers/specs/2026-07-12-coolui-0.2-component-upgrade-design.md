## 3. 组件范围

### 3.1 新增 MonthCalendar

在共享组件契约中新增第 43 个组件：

- 规范名：`MonthCalendar`
- 分类：`content`
- API kind：`dateInput`
- value type：`date`
- 事件：`select`、`monthChange`
- slots：`header`、`day`、`marker`
- 初始成熟度：四端均为 `planned`

共享日期单元模型：

```text
date                ISO 本地日期 YYYY-MM-DD
day                 公历日号
secondaryText       农历、节日或其他次要文字
accessibilityLabel  完整本地化读屏文案
isToday             是否今天
isSelected          是否选中
isDisabled          是否不可选
tone                neutral | accent | success | warning | danger
badge               休、班等短标签
markers             最多三个语义色标记
```

组件只渲染受控数据并发送事件，不计算公历、农历、节日或调休。

四端 API：

- SwiftUI：`CoolMonthCalendar` 使用 `Binding<Date>`、显示月份和 `[CoolCalendarDay]`。
- Compose：`CoolMonthCalendar` 使用 `LocalDate`、`YearMonth` 和 `[CoolCalendarDay]`。
- ArkUI：`CoolMonthCalendar` 使用 ISO 日期字符串和可序列化日期单元数组。
- 微信：`<cool-month-calendar>` 使用 `year`、`month`、`days`、`selected-date`，发送 `select`、`monthchange`。

月历只有一个共享玻璃表面，日期格使用语义色和描边，禁止每个日期格单独实时模糊。

### 3.2 微信原生 CoolButton

`cool-button` 内部从通用 `<view role="button">` 改为微信原生 `<button>`，支持：

- `label`、默认 slot、`icon` slot
- `disabled`、`loading`、`tone`、`size`、`material`
- `openType`、`formType`
- 普通 `tap`
- `getuserinfo`、`contact`、`getphonenumber`、`error`、`opensetting`、`launchapp`、`chooseavatar`、`agreeprivacyauthorization` 事件透传

加载和禁用状态阻止普通 tap。`open-type="share"`、登录和表单能力继续由微信原生按钮执行。

### 3.3 微信 TabBar 与 SegmentedControl

两个组件都使用受控接口：

```text
options: [{ value, label, disabled?, badge? }]
value: string | number
change detail: { value, index }
```

- `SegmentedControl` 用于两到五个紧凑选项。
- `TabBar` 用于页面级标签，可横向滚动并显示 badge。
- 本阶段不引入应用私有图标组件；语义图标在下一纵向切片处理。

## 4. 平台实现

### SwiftUI

- 使用 `LazyVGrid`、原生 Button、Binding 和 SwiftUI 无障碍语义。
- iOS 26.1+ 使用完整玻璃按钮材质；iOS 26.0 保留参数less 原生玻璃回退。
- 支持动态字体、降低透明度、减少动画和高对比度环境。

### Compose

- 使用受控 `YearMonth`/`LocalDate` 模型和 Compose 原生可点击语义。
- 月历共享 `CoolGlassSurface`，日期格不应用 RenderEffect。
- Android API 31+ 编译、JUnit 与 Compose UI Test 覆盖。

### ArkUI

- 使用 ArkTS 可序列化模型和 Grid 布局。
- 源码契约验证通过不等于 HAR 已验证；只有 DevEco self-hosted 构建通过后才提升成熟度。

### 微信小程序

- 新增独立 `cool-month-calendar` 自定义组件目录。
- Button、TabBar、SegmentedControl 不再由通用生成壳渲染。
- 使用基础库 3.14.3 可用的原生 button、scroll-view 和可访问性属性。
- 透明度降低时使用 `solidFallback`。

## 5. 错误处理与边界

- `days` 不是数组时显示空态。
- 日期缺少 `accessibilityLabel` 时回退到 ISO 日期和 `secondaryText`。
- disabled 日期不发送 select。
- markers 只渲染前三个。
- 月份切换只发送方向，不缓存月份。
- options 中重复 value 时首次匹配生效，并在开发校验中报告。
- Button 的开放能力错误原样透传，不显示虚假的成功状态。
