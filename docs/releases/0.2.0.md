# cooL UI 0.2.0

cooL UI 0.2.0 is a local-only alpha snapshot and is not publicly published. It expands the shared contract to 43 components while keeping every platform maturity value honest and conservative.

## Highlights

- MonthCalendar is controlled on SwiftUI, Compose, ArkUI, and WeChat, with parent-owned selection, displayed-month state, day data, and extreme-state Catalog fixtures.
- The WeChat Button now renders a native `<button>` and forwards supported native capabilities; this intentional 0.x breaking improvement changes DOM, form ownership, and event behavior from the earlier shell implementation.
- WeChat TabBar and SegmentedControl now use controlled values and request parent updates instead of mutating internal selection.
- `contracts/release.json` drives the canonical release pipeline for package versions, generated sources, Catalog labels, local artifacts, and docs.
- ArkUI ARGB channel parsing and selected-day contrast were corrected.

## Delivery and maturity

SwiftPM uses the repository checkout, Compose uses the local Maven repository, WeChat uses generated npm tarballs, and ArkUI uses a locally assembled HAR. The HarmonyOS 6 HAR build remains pending until it runs in a real DevEco environment; source validation is not presented as a successful native build.

Changesets records change intent, but the 0.2.0 versions have already been synchronized from the canonical release contract. Therefore this note does not add a pending Changesets bump that would incorrectly propose 0.3.0.
