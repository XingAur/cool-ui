## Isolated npm consumer

A release test creates a temporary snapshot from `git ls-files`, copying current tracked working-tree contents while excluding `node_modules`, `build`, and `artifacts`. It places stale sentinels in the fixture, runs `scripts/build-artifacts.mjs` there, and then uses the fixture's `examples/npm-consumer` with portable pnpm process resolution. The fixture and its temporary `node_modules` are removed in `finally`. No public registry is needed because both dependencies are local `file:` tarballs.

```sh
node --test tests/release.test.mjs
```

Within that automated command, the fixture runs offline lockfile-only installation, offline frozen installation with the hoisted linker, and `pnpm test`. The regression environment deliberately points `npm_execpath` at a fake `npm-cli.js`; portable resolution must ignore it and find pnpm through `PNPM_HOME` or `PATH`. Result: PASS. The installed consumer resolved `@cool-ui/tokens@0.2.0` and `@cool-ui/wechat@0.2.0`, then verified canonical tokens, the 43-component manifest, MonthCalendar root/default generics, native Button WXML, and controlled TabBar/SegmentedControl source. The test also fingerprints the real repository `artifacts` and `packages/wechat/dist` trees before and after the suite and requires both to remain unchanged.
